# Changelog

Все значимые изменения в проекте JAWL фиксируются в этом файле. Формат базируется на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).


## [0.9.3.1-beta] - 2026-04-29

### Added
- **Архитектурная документация**: В папку `docs/` добавлены файлы `architecture.md` (описание слоев L0-L3 и EventBus) и `memory_and_drives.md` (описание архитектуры SQL/Vector памяти и системы мотиваторов).
- **Опциональные инструкции**: Инструкции по взаимодействию с таблицами (`DRIVES.md`, `TASKS.md`, `MENTAL_STATES.md`, `PERSONALITY_TRAITS.md`) вынесены в отдельные файлы. Теперь они подтягиваются в системный промпт только если соответствующие модули включены в `settings.yaml`, что экономит токены.
- **Новые тесты**: (tavily, jawl.py, cli/, db_managers).
- **Template RPC**: Добавлен шаблон для RPC кода в `src/utils/rpc_wrapper_template.py`, избавляемся от строкового хардкода.

### Fixed
- **Терминал зависал при таймауте долгих процессов**: Исправлен баг, при котором `communicate()` зависал навсегда. Добавлено рекурсивное убийство дерева процессов (вместе с дочерними потоками вроде `node`), что закрывает дескрипторы `stdout/stderr` и корректно освобождает `asyncio`.

## [0.9.3-beta] - 2026-04-28

### Added
- **New Interface - Web Browser**: реализация асинхронного веб-браузера с cookie, созданием скриншотов, scroll, click и подобными навыками. 
- **Coordinate Grid Overlay**: В новый навык `take_screenshot` добавлен параметр `with_grid=True`. Агент теперь может накладывать полупрозрачную координатную сетку с шагом 100px поверх скриншотов веб-страниц для точного визуального позиционирования.
- **Pixel-perfect Interactions**: Добавлены навыки `click_coordinates(x,y)`, `hover_coordinates(x,y)` и `type_text(text)`. Если ARIA-дерево пустое (например, из-за антифрода или сложных `<canvas>` элементов), агент может взаимодействовать с сайтами, вычисляя пиксели по сетке.
- **Playwright Auto-install**: Если в системе установлена библиотека, но отсутствуют бинарники браузеров, клиент `WebBrowserClient` автоматически вызовет `playwright install chromium` при первом старте.

### Changed
- **Web Navigation**: Изменен механизм ожидания Playwright с `networkidle` на `load`. Это полностью устраняет падение браузера по Timeout на современных SPA-сайтах (GitHub, Ozon), которые держат бесконечные фоновые WebSocket соединения.
- **Native ARIA Snapshots**: Вырезан самописный костыль `_flatten_aom`. Фреймворк мигрировал на новый нативный метод Playwright `aria_snapshot()`, который идеально оптимизирован для AI-агентов и отдает легкую YAML-структуру страницы.

### Fixed
- Исправлена ошибка `ModuleNotFoundError`, так как зависимость `playwright` отсутствовала в requirements.

## [0.9.2.1-beta] - 2026-04-28

### Added
- **Custom Dashboards & Dynamic Context)**: Агент получил возможность создавать собственные "приборные панели" в системном промпте. Фоновые демоны могут тихо обновлять эти данные через `framework_api.update_dashboard()`, не прерывая сон агента (Heartbeat) и не расходуя лимиты API.
- **Template for `framework_api.py`**: Инъекция библиотеки в песочницу вынесена из хардкода в статический шаблон `src/utils/framework_api_template.py`. В файл добавлены подробные In-Context примеры кода и докстринги, чтобы агент мог изучать их через `read_file`.

### Changed
- Переименование `jawl_api.py`. Название `jawl_api.py` заменено на `framework_api.py`.
- Рефакторинг реестра навыков (`registry.py`). Избавились от килограмма защитного кода внутри самих скиллов.
- **Защита `framework_api.py`**: Файл моста между песочницей и ядром жестко заблокирован в Гейткипере ОС. Агент больше не сможет случайно удалить или изменить его, сломав своих же демонов.

### Fixed
- **Windows BOM Issue**: Исправлен классический баг кодировок Windows. Если пользователь редактирует `.env` или `.yaml` конфиги через стандартный Блокнот, система больше не будет игнорировать первые строки из-за невидимого символа `\ufeff` (перешли на парсинг через `utf-8-sig`).

## [0.9.2-beta] - 2026-04-28

### Added
- **Pydantic Guard Layer**: Динамическая валидация типов (Type Coercion) "на лету" для всех навыков агента. Если LLM присылает галлюцинацию (неверный тип аргумента), она получает детализированный Traceback для самообучения.
- **OS Access Decorators**: Политики доступа (RBAC) к хост-системе вынесены в изящные декораторы (`@require_access`).

### Changed
- Тотальный рефакторинг реестра навыков (`registry.py`). Избавились от килограмма защитного кода внутри самих скиллов.
- Кастомные навыки (из песочницы) теперь также поддерживают строгую типизацию через метапрограммирование `inspect.Signature`.

### Fixed
- **DuckDuckGo Rate Limits**: Переписан клиент DDG. Добавлен `asyncio.Semaphore` и экспоненциальная задержка, чтобы Cloudflare не банил агента при массовых параллельных запросах во время `DeepResearch`.

## [0.9.1-beta] - 2026-04-27

### Added
- **Система «вкладок редактора»**: Агент теперь может использовать `open_file` / `close_file`, чтобы держать актуальный код прямо в системном промпте.
- **Deploy Sessions (L2 Host OS)**: Механизм безопасной самомодификации. Система делает Copy-on-Write бэкап исходников перед правками агента.
- **Авто-валидация деплоя**: При коммите сессии автоматически запускаются `pytest` и синтаксический чек (`compileall`). В случае провала - автоматический Rollback и возврат Traceback ошибки агенту.
- **Meta Access Level 3 (CREATOR)**: Агент получил возможность превращать любые Python-скрипты из песочницы в нативные навыки через Proxy-паттерн.
- **Фоновые демоны**: Поддержка запуска долгоживущих скриптов в `sandbox/`.
- **Инъекция событий (`jawl_api.py`)**: Мини-библиотека для песочницы, позволяющая фоновым скриптам пробрасывать события и данные (вебхуки) обратно в EventBus агента.
- **Multi-window CLI**: Возможность спавнить окна логов (`--logs`) и терминала (`--terminal`) в отдельных процессах ОС. Теперь можно мониторить систему, не выходя из главного меню.
- **Навык `patch_file`**: Точечное редактирование кусков кода через Search & Replace вместо полной перезаписи файлов.

### Changed
- **Рефакторинг HostOS**: Монструозный модуль распилен на изолированные поллеры (Telemetry, Daemons, FileWatcher).
- **Web-модуль (Паттерн Стратегия)**: Движки поиска (Tavily/DDG) и чтения (Jina/Trafilatura) теперь полностью независимы и взаимозаменяемы через конфиг.
- **DuckDuckGo**: Добавлены ретраи с экспоненциальной задержкой (exponential backoff) для обхода 202 RateLimit.
- **CLI Core**: Верстка на `rich` переписана с использованием `Align.center` и `Group`. Рамки больше не "разваливаются" при изменении размера окна терминала.

### Fixed
- Оптимизация потребления токенов за счет умного кэширования дерева директорий.
- Исправлена критическая ошибка структуры конфигурации при отсутствии новых полей (добавлены Safe Defaults в Pydantic-схемы).
