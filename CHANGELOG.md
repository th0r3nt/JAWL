# Changelog

Все значимые изменения в проекте JAWL фиксируются в этом файле. Формат базируется на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).


## [0.12.2-beta] - 2026-05-03

### Added
- **Навык `run_pytest`**: В интерфейс `Host OS` добавлен легальный навык для безопасного запуска фреймворка тестирования. Теперь субагенты (QA и CODER) могут проверять архитектуру без конфликтов с жестким Гейткипером песочницы (который ранее блокировал попытки утилиты pytest прочитать системные файлы).
- **TDD Директивы (Test-Driven Development)**: В системный промпт роли `CODER` и в описание навыка `commit_deploy_session` внедрены жесткие инструкции по работе с тестами. Агент теперь осознает, что при изменении ядра фреймворка он обязан обновить соответствующие тесты, чтобы деплой прошел успешно.

### Changed
- **Изящная деградация терминала (Graceful Disconnect)**: Улучшена работа клиентской части локального CLI-терминала (`terminal_chat.py`). Теперь, если агент уходит в перезагрузку (ребут) или падает с ошибкой, терминал пользователя не крашится с `[WinError 10053] Connection lost`, а аккуратно выводит красное уведомление о разрыве связи и возвращает в главное меню.

## [0.12.1-beta] - 2026-05-03

### Added
- **Навигация для ROOT-агентов**: Теперь, если агент получает высший уровень доступа к ОС (`ROOT / 3`), в его приборную панель (`L0 State`) автоматически подтягиваются абсолютные координаты хост-машины (путь до фреймворка и домашней директории). Агенту больше не нужно угадывать расположение файлов за пределами фреймворка вслепую.
- **Покрытие тестами:** Добавлены новые unit-тесты для файлового поиска (`HostOSSearch`), метаданных файлов (`HostOSMetadata`), фоновых демонов и оберток `GraphRAG` (`VectorSearchWrapper`, `GraphSearchWrapper`).

### Changed
- **Отмена DWIM-логики путей (Угадывание отключено)**: Тотальный рефакторинг `HostOSClient._resolve_path`. Механизм "Do What I Mean" вырезан под корень во избежание шизофрении агента (когда попытки записать код в ядро приводили к созданию файлов в песочнице). Теперь **абсолютно все** относительные пути резолвятся строго от корня JAWL.
- **Поддержка ООП в RPC (Custom Skills)**: Навык `execute_sandbox_func` и обертка `rpc_wrapper.py` научились работать с классами. Теперь агент с уровнем `CREATOR` может писать ООП-скрипты и вызывать методы инстансов напрямую (передавая `func_name="MyClass.my_method"`).
- **Терминал-совместимый UI**: Вырезаны эмодзи из главного CLI-меню (`menu.py`). Они заменены на строгие и на 100% совместимые ASCII-индикаторы (`[>]`, `[■]`, `[@]`). Это устранило баг с "пустыми квадратами" на Windows консолях со скудным набором шрифтов.

### Fixed
- **Обновленный JSON-парсер**: Полностью переписаны методы `_parse_response` в главном Оркестраторе (`ReactLoop`) и цикле субагентов (`SubagentLoop`). Раньше агент уходил в ошибку, если модель писала `*(thoughts)*` вне Markdown-блоков. Теперь парсер просто ищет первую `{` и последнюю `}`, беспощадно игнорируя любой мусор до и после JSON'а.
- **Config Override Vulnerability**: Закрыта критическая дыра в безопасности уровней доступа (SANDBOX / OBSERVER / OPERATOR). Добавлен жесткий ReadOnly-лок в `Gatekeeper` на директорию `config/`. Агент больше не сможет переписать `settings.yaml` своими файловыми навыками, чтобы обнулить политики безопасности (доступно только для `ROOT`).
- **VectorSearchWrapper:** Исправлен баг, из-за которого при дедупликации семантических векторов (если одна и та же запись была найдена и в знаниях, и в мыслях) обновлялся только `score`, но не перезаписывалась метаинформация (коллекция, теги).
- **Гейткипер в тестах:** Исправлены пути во всех навыках, обращающихся к песочнице, для их совместимости с отключенной DWIM-логикой и прохождения тестов безопасности.

## [0.12.0-beta] - 2026-05-02

### Added
- **Vector-Graph RAG (Гибридный механизм контекстной памяти):** Полностью интегрирована подсистема графовой памяти на базе `KuzuDB`. Теперь RAG агента работает по принципу глубокого семантического резолвинга: 
  - Оркестратор динамически дробит мысли агента на семантические куски (Vector Chunking).
  - Сверхбыстрый извлекатель `EntityExtractor` (на алгоритме Aho-Corasick через `FlashText` и `RapidFuzz` для морфологии русского языка) находит упоминания существующих графовых сущностей в сыром тексте.
  - Извлеченные факты из `Qdrant` подтягивают за собой связанные узлы из графа, а описание графовых узлов инициирует новые запросы к векторной БД.
  - Агент получил возможность физически связывать сущности в памяти (Graph Edges), выстраивая причинно-следственные связи.
- **Системный промпт (INSTRUCTIONS):** Агент обучен работе с графовой базой (ему объяснено, зачем и как связывать сущности для создания Knowledge Graph).
- **Интеграционные тесты (Хардкор):** Написаны фундаментальные тесты, проверяющие работу системы в реальных условиях:
  - *GraphRAG:* e2e полигон, симулирующий одновременное обращение к `Qdrant` и `KuzuDB`.
  - *Deploy Sessions:* физическая проверка механизма отката кода (Rollback), если агент закоммитил скрипт с синтаксической ошибкой.
  - *Swarm Concurrency:* проверка Semaphore (гарантия, что при вызове 5 субагентов, ОС и LLM API не умрут от перегрузки, так как параллельно работают только N воркеров).
  - *WebHooks:* реальное поднятие aiohttp-сервера на случайном порту и прием физического HTTP POST запроса.
- **Новая документация:** Написан исчерпывающий Markdown-мануал `docs/yaml/settings/rag.md`, объясняющий логику работы и конфигурацию гибридного поиска, плюс обновлен `README.md`.

### Changed
- **Очистка юнит-тестов RAG:** Старые тесты (`test_rag_memories.py`) были полностью переписаны и адаптированы под архитектуру инъекции DI-зависимостей `GraphManager` и новой структуры `RAGConfig`.

### Fixed
- **Smart Bootstrapper Deep Check:** Устранен баг "состояния гонки" (Race Condition) при обновлении зависимостей. Теперь `jawl.py` принудительно делает `import src.main`, чтобы обнаружить отсутствие скрытых L1/L2 библиотек (вроде `rapidfuzz`) до запуска меню и корректно триггернуть авто-установку из `requirements.txt`.
- **Deploy Rollback Crash:** Исправлено падение механизма самовосстановления агента. Ранее системная утилита `compileall` создавала папки `__pycache__`, которые крашили `shutil.copy2` при попытке вернуть старый код. Теперь весь `.pyc` мусор игнорируется, а родительские директории создаются безопасно.
- **Извлечение коротких аббревиатур (EntityExtractor):** Исправлено условие длины слова с `> 2` на `>= 2`. Теперь экстрактор корректно находит в тексте и подтягивает графовые узлы из двух букв (например, "ИИ", "IT", "OS").
- **Кириллица в RapidFuzz:** Исправлен баг, из-за которого регулярное выражение границ слова (`\b`) некорректно обрабатывало русские аббревиатуры. Перешли на безопасное извлечение слов через `re.findall(r'\w+', ...)`.
- **Тесты Swarm:** Исправлен мок ответа LLM в интеграционном тесте роя (`test_integration_swarm_full_cycle`). Ответ приведен в строгое соответствие со схемой `execute_skill` (добавлены `thoughts` и `actions`), что устранило падение парсера.

## [0.11.0-beta] - 2026-05-01

### Added
- **Документация интерфейсов:** В директорию `src/l2_interfaces/` добавлен эталонный шаблон-пример (`_interface_example/`) с подробными инженерными комментариями. Шаблон обучает разработчиков архитектурным принципам JAWL (SOLID, DI через `bootstrap`, изоляция I/O в `client`, поллинг в `events` и разделение ответственности в `skills/`), снижая порог входа для создания кастомных коннекторов.
- **Security Tests:** Внедрено жесткое тестирование `Gatekeeper` (Host OS). Полное покрытие уязвимостей Path Traversal, попыток кражи секретов из `.env` и несанкционированной записи в системные файлы (`framework_api.py`) для всех уровней доступа (SANDBOX, OBSERVER, OPERATOR, ROOT).
- **Hardcore Integration Tests:** Добавлены прагматичные интеграционные тесты для проверки реального (без моков) взаимодействия слоев системы:
  - *RPC Sandbox:* полный цикл написания скрипта агентом в песочнице и динамического вызова его функций.
  - *Auto-RAG:* физическое извлечение фактов из локальной векторной базы (Qdrant) и инъекция в контекст.
  - *EventBridge & Heartbeat:* экстренное пробуждение агента через публикацию в реальную шину событий.
  - *FileWatcher Diff:* генерация git-подобных diff-ов при физическом изменении файлов на диске.
- **Roadmap:** Создан официальный `TODO.md` с долгосрочными планами на новые фичи.
- **Zero-Day Sandbox Escape Patch:** Устранена критическая уязвимость, позволявшая Python-скриптам в песочнице (через навыки `execute_script` и `execute_sandbox_func`) обходить Гейткипер Host OS. Ранее скрипты могли читать файлы вне песочницы (Path Traversal через стандартный `open()`), получать доступ к токенам через `os.environ` и выходить в Shell через `subprocess`. Внедрена инъекция `sandbox_runner.py`, которая "на лету" переопределяет `builtins.open`, блокирует модули `os` и `subprocess` для дочерних процессов, а также жестко вычищает (скраббит) секреты из переменных окружения.

### Changed
- **Разгрузка инициализатора системы (`main.py`):** Главный файл оркестратора избавлен от хардкода. Логика сборки слоев (DI-контейнера) вынесена в паттерн Строитель (`src/builder.py`), а сложная подписка на EventBus инкапсулирована в `EventBridge` (`src/utils/event/bridge.py`). `main.py` стал чистым, легко читаемым (почти) фасадом.
- **Масштабный рефакторинг Host OS (Устранение God Object):** Гигантский модуль `files.py` распилен на 7 независимых микро-классов согласно принципу единой ответственности. Теперь логика строго разделена на `reader`, `writer`, `editor` (для патчинга), `search`, `archive`, `workspace` (вкладки агента) и `metadata`.
- **Разделение L0 State (SRP):** Монолитный файл состояний интерфейсов (`src/l0_state/interfaces/state.py`) был уничтожен. Теперь каждое состояние инкапсулировано в свой независимый файл (`github_state.py`, `email_state.py`, `calendar_state.py` и т.д.), что радикально улучшает читаемость и поддержку кода.
- **Оптимизация Gatekeeper (`HostOSClient`):** Метод `validate_path` рефакторизован. Математика вычисления физических путей (`_resolve_path`) и проверки политик безопасности RBAC (`_check_security_policies`) вынесены в отдельные приватные методы, что исключает риск случайных уязвимостей (Path Traversal).
- **Очистка Telethon Events:** Монолитный метод `_update_state`, отвечающий за парсинг диалогов и сборку дашборда, разбит на логические чанки (`_process_user_dialog`, `_process_group_dialog` и др.). Улучшена читабельность логики кэширования для обхода `FloodWaitError`.

### Fixed
- **Terminal Double-Events:** Исправлен баг, при котором отправка многострочных (с переносом `\n`) сообщений пользователем в локальном терминале приводила к генерации нескольких событий `HOST_TERMINAL_MESSAGE`. Протокол общения между клиентом и TCP-сервером переведен на строгий JSON Lines.
- **UI Markdown Rendering:** Исправлено "схлопывание" переносов строк в терминале (особенность парсера библиотеки `rich.Markdown`). Теперь маркированные списки, код и абзацы от агента рендерятся идеально ровно.
- **Silent Swarm Crashes:** Исправлено молчаливое "падение" субагентов. Ранее, если API модели было недоступно (HTTP 404/500) или субагент превышал лимит шагов (`max_steps`), его цикл просто прерывался, и главный Оркестратор ждал отчет бесконечно. Теперь система принудительно перехватывает такие краши и от лица субагента отправляет отчет с Traceback-ом ошибки главному агенту.

## [0.10.0-beta] - 2026-04-30

### Added
- **Major Feature: Подсистема Swarm (Субагенты)**: Главный агент теперь может делегировать сложные и объемные задачи специализированным фоновым субагентам. Они работают параллельно, не блокируя основной ReAct-цикл, и присылают подробные Markdown-отчеты по завершении.
- **Новые системные роли (Roles)**:
  - `CODER` (Software Engineer): Имеет доступ к файловой системе, локальному Git (clone, checkout, commit, push) и выполнению скриптов. Идеален для рефакторинга и фикса багов.
  - `WEB_RESEARCHER` (OSINT Analyst): Имеет доступ к поисковикам и чтению веб-страниц (Deep Research). Отлично подходит для сбора информации и фактчекинга.
- **RBAC для навыков (Role-Based Access Control)**: Декоратор `@skill` расширен аргументом `swarm_roles`. Субагенты видят в своем системном промпте и могут вызывать только те инструменты, которые явно разрешены их роли (принцип наименьших привилегий).
- **SubagentLoop (Stateless ReAct)**: Специализированный, облегченный цикл рассуждений для субагентов. Имеет собственный механизм дампа контекста, строгий лимит шагов (`max_steps`) и локальную историю для экономии токенов.
- **Concurrency Control**: Запуск субагентов контролируется через `asyncio.Semaphore` (`max_concurrent_workers` в настройках), чтобы защитить API ключи от жестких Rate Limits при спавне целого роя агентов.
- **EventBus Integration**: Добавлено системное событие `SUBAGENT_TASK_COMPLETED` (уровень HIGH). Когда субагент сохраняет отчет, главный агент мгновенно просыпается для его анализа.
- Настройки Swarm (включение, выбор дешевой модели и лимиты воркеров) добавлены в `settings.yaml`.

### Changed
- Тотальный рефакторинг глобального реестра навыков (`src/l3_agent/skills/registry.py`) для поддержки сохранения и фильтрации объектов `SubagentRole`.
- Ядро `System` (`src/main.py`) обновлено: инициализация `SwarmManager` и специфичного навыка `SubagentReport` происходит динамически при включенном параметре в конфигурации.

## [0.9.4-beta] - 2026-04-29

### Added
- **New Interface - Web Hooks**: Реализован полноценный HTTP-сервер на базе `aiohttp` для приема вебхуков. Позволяет агенту интегрироваться с любыми внешними сервисами (GitHub Actions, Stripe, Smart Home, Cron-скрипты) без написания коннекторов.
