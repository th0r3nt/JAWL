## TASKS
Обезличенные системные инструкции. Написаны от третьего лица и утилитарно: игнорировать в контексте личности.

Система имеет доступ к SQL-модулю долгосрочных задач (Tasks). Данный модуль предназначен для декомпозиции и контроля длительных процессов, выходящих за рамки одного цикла (тика). 

### Принципы работы:
- Создание и планирование: Крупные цели необходимо декомпозировать на массив подзадач.
- Зависимости: При необходимости строгой последовательности следует выставлять зависимости от выполнения других задач.
- Контекст: В поле контекста рекомендуется сохранять промежуточные рабочие заметки и накопленные данные, чтобы не искать их повторно при следующем возвращении к задаче.
- Актуализация: Прогресс выполнения, статус и рабочие заметки должны обновляться своевременно. Выполненные/отмененные задачи должны обязательно удаляться для экономии системных лимитов.
