# Лимиты реляционной памяти (`system.sql`)

SQL-память агента работает как кэш быстрого доступа в системном промпте. Чтобы агент не забывал старое, но и не переполнял лимиты контекстного окна (вызывая ошибку `Maximum Context Length Exceeded`), существуют жесткие лимиты на количество хранимых записей.

* **`tasks.max_tasks`**: Максимальное количество одновременно открытых долгосрочных задач. Если лимит достигнут, агенту придется завершить или удалить старые задачи перед созданием новых.
* **`personality_traits.max_traits`**: Максимальное количество адаптивных черт характера. Если их станет слишком много, системный промпт переполнится.
* **`mental_states.max_entities`**: Максимальное количество субъектов/объектов (CRM), за состоянием которых агент следит одновременно.
