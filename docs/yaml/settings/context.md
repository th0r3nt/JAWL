# Управление контекстом (`context_depth`)

Каждое выполненное действие агента логируется в базе данных как "Тик" (Мысли -> Вызов функции -> Результат). Агент видит свою историю в системном промпте, что позволяет ему сохранять нить повествования.

Для экономии токенов история разделена на два формата:
1. **Detailed Ticks (`detailed_ticks`)**: Самые свежие шаги. Передаются агенту с максимальной детализацией (включают полные ответы функций и JSON-параметры).
2. **Short Ticks**: Старые шаги. Передаются агенту в сжатом, агрегированном виде. Длинные результаты обрезаются, а ответы функций сокращаются.

**Пример (`ticks: 15`, `detailed_ticks: 3`)**: Агент будет помнить последние 15 действий. Из них 12 будут сжатыми (чтобы не сжечь лимиты токенов на старую информацию), а 3 последних будут полными (чтобы иметь максимальный контекст для текущей задачи).
