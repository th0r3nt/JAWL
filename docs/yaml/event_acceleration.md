# Event Acceleration (Множители пробуждения)
Архитектура JAWL базируется на экономии вычислительных ресурсов: в отсутствие задач агент находится в режиме ожидания (Heartbeat-сне). Параметр `heartbeat_interval` определяет максимальное время (в секундах) между пробуждениями. 
Система `Event Acceleration` определяет, насколько сильно входящие события (внешние триггеры) сокращают текущий сон агента, заставляя его просыпаться раньше расписания.

## Математика множителей

Каждое событие (от `CRITICAL` до `BACKGROUND`) имеет свой числовой множитель.
**Формула:** `Новое_время_до_пробуждения = Текущий_остаток_сна * Множитель_события`

### Пример расчетов:
Допустим, `heartbeat_interval` равен 600 секундам (10 минут).
Агент ушел в сон. Прошло 2 минуты, осталось спать 8 минут.
Поступает событие уровня `MEDIUM` (например, упоминание в группе Telegram), для которого установлен множитель `0.6`.
- *Результат:* 8 минут * 0.6 = 4.8 минуты. Агент проснется не через 8 минут, а через 4.8. Сон был сокращен на 40%.

Если сразу после этого поступит событие уровня `LOW` (множитель 0.8), остаток в 4.8 минуты умножится на 0.8, оставив 3.84 минуты до пробуждения.

### События CRITICAL (Экстренное прерывание)
Уровень `CRITICAL` по умолчанию имеет множитель `0.0`.
Это особый системный уровень. Если множитель события равен нулю или близок к нему (< 0.01), система действует иначе:
1. Время сна сбрасывается в ноль (пробуждение происходит моментально).
2. Если агент в данный момент **не спал**, а находился в процессе вычислений (ReAct-цикл), текущий процесс принудительно прерывается.
3. Агент моментально перезапускается с новым, чистым контекстом, где главным триггером будет указано поступившее критическое событие.
