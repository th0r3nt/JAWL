# 📝 JAWL Roadmap & TODO

Добро пожаловать в официальный трекер задач фреймворка JAWL.
Здесь собраны планы по развитию архитектуры, новые фичи и технический долг.

---

## 🚀 Высокий приоритет (Ближайшие релизы)

- [ ] **Графовая база данных (Knowledge Graph) & GraphRAG**
  - Интеграция `KuzuDB` (или аналога) рядом с Qdrant и SQLite.
  - Научить агента строить связи между сущностями (Mental States) и концепциями ("A связано с B").
  - Внедрить гибридный RAG: объединение векторного поиска по семантическому сходству с графовым проходом по связям. Это превратит память агента в понимание.

## 🧠 R&D

- [ ] **Tree of Thoughts (MCTS)**
  - Внедрение алгоритма симуляции мыслей перед выполнением действий (Монте-Карло поиск в дереве).
  - Настройка конфигурации (`tree_of_thoughts: true`) с возможностью выбора выделенной LLM и интервала активации.
  - Позволит агенту предвидеть последствия выполнения опасных команд (ROOT-доступ) на несколько шагов вперед, резко снизив количество "импульсивных" ошибок.

## 🧹 Технический долг и Рефакторинг

- [ ] **Докстринги для тестов** *(неизбежный ад)*
  - Пройтись по всей директории `tests/` и добавить исчерпывающие описания к каждому `test_...` методу. Да, это больно. Но инквизиция чистого кода требует жертв.
- [ ] **Оптимизация зависимостей**
  - Провести строгую ревизию `requirements.txt` на предмет транзитивных и неиспользуемых библиотек.

## 💡 Бэклог (Идеи на будущее)

- [ ] **Голосовой интерфейс (TTS / STT)**
  - Полноценная интеграция микрофона и динамиков (`Whisper` + `Edge TTS`/`ElevenLabs` + любые другие реализации), чтобы с агентом можно было общаться голосом прямо на хост-машине.
- [ ] **Telegram UI Dashboard**
  - Сделать WebApp внутри Telegram (Mini App) для визуального управления ресурсами и базами данных агента, чтобы не заходить в консоль хоста.
- [ ] **Discord Interface**
  - Добавить L2-интерфейс для работы с Discord API (чтение серверов, отправка сообщений, управление ролями).
