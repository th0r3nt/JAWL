# JAWL (Just A While Loop)

**JAWL** - это stand-alone фреймворк для создания автономных ИИ-агентов непрерывного цикла. 

Проект построен вокруг простой концепции: агент функционирует в рамках бесконечного ReAct-цикла (Reasoning and Acting). Он непрерывно собирает контекст, формирует цепочку мыслей (Chain of Thought), выполняет полезные действия через доступные интерфейсы и уходит в сон до следующего тика или внешнего триггера.

Никакого Docker. Фреймворк спроектирован для нативного запуска на хост-машине с жестким контролем доступа (Gatekeeper), чтобы агент мог напрямую взаимодействовать с ОС, файловой системой и локальными сетями.

## Для чего идеально подойдет этот фреймворк?

* **DevOps/SysAdmin Bot**: агент, который живет на сервере, мониторит логи, чистит кэши, сам ищет решения ошибок в интернете и пишет вам в Telegram.
* **OS-level Personal Assistant**: помощник на личном компьютере (управление файлами, поиск документов, автоматизация рутины).
* **Research Agent**: агент, которому дается тема на ночь, и он до утра сёрфит интернет (DuckDuckGo), складывает факты в векторную БД (Qdrant) и утром выдает готовый отчет.


## 🏗 Архитектура (L0 - L3)

Проект строго разделен на 4 зоны ответственности:

* **L0 (State):** Пассивные витрины данных (MRU-кэши). Хранят снимки состояния интерфейсов (последние чаты, метрики ОС, историю браузера). Экономят контекст и предотвращают дорогие API-вызовы.

* **L1 (Databases):** Локальный слой памяти. 
  * *SQL (SQLite):* Долгосрочные задачи, приобретенные черты характера, отслеживание сущностей (Mental States) и полные логи тиков агента.
  * *Vector (Qdrant + FastEmbed):* Семантическая память. Работает локально (CPU). Разделена на коллекции `knowledge` (факты) и `thoughts` (рефлексия).

* **L2 (Interfaces):** Окна во внешний мир.
  * *Host OS:* Управление файлами, процессами, сетью и терминалом. Встроенная песочница (`sandbox/`).
  * *Telegram:* Поддержка User-API (`Telethon`) и Bot-API (`Aiogram`).
  * *Web Search:* Парсинг веб-страниц и поиск (DuckDuckGo).
  * *Meta:* Интерфейс для изменения конфигурации агента прямо в рантайме.

* **L3 (Agent Core):** Мозг системы. Содержит `ReactLoop`, сборщики промптов/контекста, автоматический RAG и `Heartbeat`.


## ⚙️ Ключевые особенности

* **Event-Driven Heartbeat:** Агент не тратит токены впустую. Он спит между тиками, но мгновенно просыпается при входящих событиях через `EventBus`. Оставшееся время сна динамически сокращается в зависимости от уровня события (CRITICAL, HIGH, MEDIUM, LOW, BACKGPRUND).
* **Access Level (Gatekeeper):** Встроенная система прав доступа к Host OS. 4 уровня: от полной изоляции в папке `sandbox/` до `ROOT` прав над всей ОС хоста. Защита файлов `.env` на уровне кода.
* **API Key Rotator:** Автоматическая ротация пула LLM-ключей. Учитывает Rate Limits (429) и перманентные баны (401), прозрачно переключая сессии.
* **Stand-alone:** Никаких контейнеров, сложной инфраструктуры или облачных баз данных. Зависимости устанавливаются через `pip`, векторная БД поднимается прямо в папке проекта.
* **Строгие тесты:** для проекта написано порядка 130+ тестов для проверки работоспособности и стабильности кода.


## 🚀 Установка и запуск


1. **Клонирование и зависимости:**
   ```bash
   git clone https://github.com/th0r3nt/JAWL.git
   cd jawl.py
   python -m venv venv
   source venv/bin/activate  # Для Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

2. **Конфигурация:**
   * Скопируйте `.env.example` в `.env` и добавьте ваши LLM API ключи (и токены Telegram, если нужны).
   * При необходимости отредактируйте `config/settings.yaml` (настройки модели, БД, лимитов) и `config/interfaces.yaml` (уровни доступа и включенные модули).

3. **Запуск:**
   ```bash
   python jawl.py
   ```

## 📜 Принципы разработки
* **KISS & DRY:** код пишется так, чтобы его можно было читать без словаря.
* **No overengineering:** абстракции вводятся только там, где они решают реальную проблему, а не ради красивого паттерна.
* **SOLID:** строгая изоляция слоев. L3 не знает, как работает L1, он работает только с интерфейсами.
