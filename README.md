# Цифровой раб

Задумка: создать отдельный мини-проект как-нибудь на выходных.
В основе него лежит сверх-быстрая и дешевая Gemini 3.1 Flash Lite, которая может отвечать за 1-3 секунды, при этом являясь крайне дешевой.

Ядро проекта - бесконечный цикл ReAct, в котором: 
1. Скидывается изображение текущего экрана;
2. Делается запрос к нейросети (с тулзами по типу click, scroll и прочее);
3. Производятся действия;
4. Смотреть на пункт 1.

Таким образом, этот проект - реализация цифрового работника, которому можно давать человеческие задачи.
Предполагается, что у этого сверх быстрого работника будут свои базы данных - чтобы хранить контекст и важную информацию по типу "папка с мемами пользователя лежит в ...".
Агент может производить любые действия в системе, смотря на экран - работая как настоящий человек.

Однако, LLM ужасно определяют точные пиксели. 
Если скинуть ей скриншот 1920x1080 и сказать "Кликни на иконку Telegram", 
она выдаст координаты от балды (например, x: 500, y: 100), и кликнет мимо.

* Обязательно реализовать: 
Set-of-Mark (SoM): перед тем как отправить скриншот в Gemini, 
скриптом (через Python) накладываем поверх скриншота полупрозрачную сетку с цифрами 
(например, матрицу 10x10, где каждый квадрат пронумерован от 1 до 100).

Пример ответа нейросети:
{
  "thought": "Вижу открытый браузер. В строке поиска ничего нет. Нужно ввести запрос.",
  "actions": [
    {"tool": "mouse_move", "grid_id": 45},
    {"tool": "mouse_click", "button": "left"},
    {"tool": "keyboard_type", "text": "как удалить репозиторий github"},
    {"tool": "keyboard_hotkey", "keys": ["enter"]}
  ]
}


А также: нужно в каждый инструмент добавить параметр "задержка", чтобы LLM могла делать несколько действий последовательно.
Без этого параметра цепочка дейсвий будет выполняться моментально, что может нарушить последовательность.
