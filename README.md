# JAWL (Just A While Loop)

**JAWL** - это stand-alone фреймворк для создания автономных ИИ-агентов непрерывного цикла. 

Проект построен вокруг простой концепции: агент функционирует в рамках бесконечного ReAct-цикла (Reasoning and Acting). Он непрерывно собирает контекст, формирует цепочку мыслей (Chain of Thought), выполняет полезные действия через доступные интерфейсы и уходит в сон до следующего тика или внешнего триггера.

Никакого Docker и облачных баз данных. Фреймворк спроектирован для нативного кроссплатформенного запуска на хост-машине с жестким контролем доступа (Gatekeeper), чтобы агент мог безопасно взаимодействовать с ОС, файловой системой и локальными сетями.

## 💡 Для чего идеально подойдет этот фреймворк?

* **DevOps / SysAdmin Bot:** Агент, который живет на сервере, мониторит процессы (RAM/CPU), читает логи, сам ищет решения ошибок в интернете и пишет вам в Telegram.
* **OS-level Personal Assistant:** Помощник на локальном ПК (управление файлами, поиск документов, автоматизация рутины).
* **Research Agent:** Агент, которому дается тема на ночь, и он до утра проводит параллельный Deep Research в сети (DuckDuckGo + Trafilatura), складывает факты в векторную БД (Qdrant) и утром выдает готовый отчет.

## 🏗 Архитектура (L0 - L3)

Проект строго разделен на 4 зоны ответственности (SOLID):

* **L0 (State):** Пассивные витрины данных (MRU-кэши). Хранят снимки состояния (последние сообщения, метрики ПК, историю браузера, таймеры). Экономят токены, так как агенту не нужно тратить API-вызовы, чтобы просто "оглядеться".

* **L1 (Databases):** Локальный слой памяти. 
  * *SQL (SQLite):* Модули памяти - долгосрочные задачи (Tasks), приобретенные черты характера (Traits), отслеживание сущностей (Mental States), мотивация (Drives) и полные логи тиков агента.
  * *Vector (Qdrant + FastEmbed):* Семантическая память. Работает локально на CPU. Разделена на коллекции `knowledge` (факты из внешнего мира) и `thoughts` (внутренняя рефлексия).

* **L2 (Interfaces):** Окна во внешний мир.
  * *Host OS:* Управление файлами, процессами и сетью на локальной хост-машине. Встроенная песочница (`sandbox/`).
  * *Telegram:* Поддержка User-API (`Telethon`) и Bot-API (`Aiogram`).
  * *Web Search:* Парсинг веб-страниц, поиск и навык `Deep Research`.
  * *Calendar:* Управление временем, установка интервальных и разовых триггеров пробуждения.
  * *Meta:* Интерфейс для изменения конфигурации агента в рантайме (перезапускать скрипт каждый раз - это для слабаков).

* **L3 (Agent Core):** Мозг системы. Содержит `ReactLoop`, сборщики контекста, автоматический RAG и `Heartbeat` (событийный пульс).

## ⚙️ Ключевые особенности

* **Smart Bootstrapper:** Забудьте про ручную возню с окружениями. Скрипт `jawl.py` сам создаст `venv`, установит зависимости из `requirements.txt` и запустит красивый CLI-интерфейс. Если вы случайно удалите зависимость, система сама восстановит ее при следующем запуске.
* **Event-Driven Heartbeat:** Агент не тратит токены впустую. Он спит между тиками, но мгновенно просыпается при входящих событиях через `EventBus`. Оставшееся время сна динамически сокращается в зависимости от уровня события (CRITICAL, HIGH, MEDIUM, LOW, BACKGROUND).
* **Gatekeeper (Access Level):** Встроенная система прав доступа к Host OS. 4 уровня: от полной изоляции в папке `sandbox/` до `ROOT` прав над всей ОС хоста. Автоматическая защита файлов `.env` на уровне кода.
* **Drives (Мотиваторы):** Агент обладает математически просчитываемой мотивацией (Дефицит от 0 до 100%). Если нет внешних раздражителей, растущий дефицит мотивации (например, "Любопытство") заставит агента самостоятельно искать себе полезную работу (в отличие от создателя фреймворка).
* **API Key Rotator:** Автоматическая ротация пула LLM-ключей. Прозрачно обходит Rate Limits (429) и перманентные баны (401), переключая HTTP-сессии на лету.

## 🚀 Установка и запуск

*Требования: Убедитесь, что у вас установлен Python 3.10+ и он добавлен в PATH.*

1. **Клонирование:**
   ```bash
   git clone https://github.com/th0r3nt/JAWL.git
   cd JAWL
   ```

2. **Конфигурация:**
   * Скопируйте `.env.example` в `.env` и добавьте ваши LLM API ключи (и токены Telegram, если планируете их использовать). Впрочем, если вы забудете, CLI-установщик спросит ключ при первом запуске.
   * При необходимости отредактируйте `config/settings.yaml` (настройки модели, БД, лимитов) и `config/interfaces.yaml` (модули и уровни доступа).

3. **Магия одной команды:**
   ```bash
   python jawl.py
   ```
   *Скрипт автоматически развернет виртуальное окружение, скачает библиотеки и откроет интерактивное меню настройки и управления агентом.*


> **🐧 Инструкция для чистого Linux (Ubuntu/Debian VPS):**
> 1. Установите системные зависимости: `sudo apt install -y python3 python3-venv git tmux`
> 2. Обязательно добавьте Swap-файл (минимум 2GB), чтобы система не убила векторную БД при инициализации.
> 3. Запускайте фреймворк внутри `tmux`, чтобы агент продолжил работать после закрытия SSH-сессии: 
>    `tmux new -s jawl` -> `python3 jawl.py`


## 📜 Принципы разработки
* **KISS & DRY:** Код пишется так, чтобы его можно было читать без словаря.
* **No overengineering:** Абстракции вводятся только там, где они решают реальную проблему. Инквизиция ООП нас за это не осудит.
* **SOLID:** Строгая изоляция слоев. Мозг (L3) не знает, как работает база данных (L1), он общается с ней только через четко очерченные интерфейсы.
