# JAWL (Just A While Loop)

**JAWL** - это stand-alone фреймворк для создания автономных ИИ-агентов непрерывного цикла. 

Проект построен вокруг простой концепции: агент функционирует в рамках бесконечного ReAct-цикла (Reasoning and Acting). Он непрерывно собирает контекст, формирует цепочку мыслей (Chain of Thought), выполняет полезные действия через доступные интерфейсы и уходит в сон до следующего тика или внешнего триггера.

🧠 **Ультимативный гибридный RAG (Vector-Graph RAG):** Одно из главных отличий фреймворка - инновационная подсистема памяти, навсегда решающая проблему "амнезии" языковых моделей на длинных дистанциях. Система сплетает классический векторный поиск по семантике (Qdrant + FastEmbed) с жесткими причинно-следственными связями из графа знаний (KuzuDB). При любом входящем событии оркестратор "на лету" извлекает сущности из текста, рекурсивно кросс-резолвит их по обеим базам и инжектит в промпт агента безупречно точную выжимку фактов и их логических связей. Этот механизм реализует человеческую ассоциативную память.

Никакого Docker и облачных баз данных. Фреймворк спроектирован для нативного кроссплатформенного запуска на хост-машине с жестким контролем доступа (Gatekeeper), чтобы агент мог безопасно взаимодействовать с ОС, файловой системой и локальными сетями.


## 💡 Для чего идеально подойдет этот фреймворк?
Благодаря архитектуре L0-L3, богатому набору интерфейсов и подсистеме субагентов, на базе JAWL можно развернуть агентов абсолютно разного профиля. Теперь ваш агент - это не просто одиночный воркер, а полноценный Оркестратор, способный управлять роем субагентов:

* **Lead Tech / Open-Source Maintainer:** Агент, живущий в ваших репозиториях на GitHub. Заметив сложный Issue, он не тратит свой дорогой контекст, а делегирует задачу фоновому субагенту `CODER` (на дешевой и быстрой `Gemini Flash Lite`). Субагент клонирует код в песочницу, дебажит его и делает Pull Request, а Главный агент смотрит на результат, ревьюит и в конце концов принимает работу, отчитываясь вам в Telegram.
* **SRE/DevOps Sentinel:** Watchdog на вашем VPS. Непрерывно мониторит потребление RAM/CPU и сетевые сокеты. Если критический сервис падает, главный агент спускает субагента для парсинга гигантских простыней `stderr` логов и поиска ошибки в сети, пока сам применяет экстренные фиксы через shell.
* **OSINT / Deep Research Syndicate:** Вы даете агенту сложную тему. Он декомпозирует её и спавнит рой субагентов `WEB_RESEARCHER`. Они параллельно шерстят интернет, вычитывают десятки сайтов, обходят мусорные SEO-статьи и приносят готовые выжимки. Главный агент консолидирует их труды, укладывает факты в семантическую память (Vector DB) и выдает вам структурированный Markdown-отчет.
* **Autonomous Self-Improving Agent:** Самый продвинутый тип агента. За счет Meta Access Level 3 (CREATOR) и универсальной функции `execute_skill` он получает полную свободу действий. Такой агент умеет:
    * Анализировать собственные "следы" работы (логи, тики, контекст).
    * Находить и исправлять ошибки в собственном промпте и алгоритмах, делегируя задачи субагентам.
    * Создавать новые навыки на лету, комбинируя вызовы старых API и библиотек, обновляя контекст: само-адаптация.
    * Экспериментировать с разными LLM, самостоятельно выбирая оптимальную модель для текущей задачи.


## 🏗 Архитектура (L0 - L3)
Проект строго разделен на 4 зоны ответственности:

* **L0 (State):** Пассивные витрины данных (MRU-кэши). Хранят снимки состояния (последние сообщения, метрики ПК, историю браузера, таймеры). Экономят токены, так как агенту не нужно тратить API-вызовы, чтобы просто "оглядеться". **Поддерживает динамические Custom Dashboards**.

* **L1 (Databases):** Локальный слой памяти. 
  * *SQL (SQLite):* Долгосрочные задачи (Tasks), приобретенные черты характера (Traits), отслеживание сущностей (Mental States), мотивация (Drives) и полные логи тиков агента.
  * *Vector (Qdrant + FastEmbed):* Семантическая память. Работает локально на CPU. Разделена на коллекции `knowledge` (факты из внешнего мира) и `thoughts` (внутренняя рефлексия).
  * *Graph (KuzuDB):* Граф знаний (Knowledge Graph). Хранит узлы и выстраивает строгие причинно-следственные связи для идеального семантического резолвинга и предотвращения амнезии на длинных дистанциях.

* **L2 (Interfaces):** Окна во внешний мир.
  * *Host OS:* Управление файлами, процессами, сетью и GUI (буфер обмена, уведомления) на хост-машине. Встроенная песочница (`sandbox/`) с защищенным API для скриптов.
  * *Telegram:* Поддержка User-API (`Telethon`) и Bot-API (`Aiogram`).
  * *GitHub:* REST API (Issues, PR, поиск трендов) + Локальный Git (агент может клонировать репозитории в песочницу, писать код и делать push/commit от своего имени).
  * *Email:* Чтение, отправка и сортировка почты (IMAP/SMTP) с автоматическим определением серверов.
  * *Web Search, HTTP, Browser & Webhooks*: Поиск, парсинг страниц, headless-браузер на базе Playwright и **собственный HTTP-сервер для приема входящих вебхуков**.
  * *Calendar:* Управление временем, установка интервальных и разовых триггеров пробуждения.
  * *Multimodality:* Зрение. Агент может "смотреть" на локальные изображения или скриншоты браузера, если основная LLM это поддерживает.
  * *Meta:* Интерфейс для изменения конфигурации агента в рантайме.
  * *Code Graph:* Инструмент Agentic Introspection. Парсит AST-дерево Python проектов, строит граф зависимостей/связей файлов, функций и классов, позволяет агенту семантически искать код и вычислять Blast Radius при рефакторинге. Агент может найти нужный кусок кода, просто описав его смысл (например: *"где сохраняются логи?"*).

* **L3 (Agent Core):** Мозг системы. Содержит `ReactLoop`, сборщики контекста, автоматический RAG, `Pydantic Guard Layer` и `Heartbeat` (событийный пульс). LLM может быть как облачной, так и локальной.

## 🐝 Swarm (Мульти-агентная подсистема)
JAWL поддерживает масштабирование через делегирование. Главный агент (Оркестратор) может спавнить фоновых **субагентов** для параллельного выполнения ресурсоемких задач.

### Зачем это нужно?
* **Экономия токенов и денег:** Главному агенту (например, дорогому `Claude Opus`) не нужно тратить свой контекст на скроллинг десятков сайтов в поисках нужной информации. Он поручает эту задачу субагенту, работающему на быстрой и дешевой модели (например, `Gemini Flash Lite`), и получает только готовую выжимку.
* **Изоляция:** Субагенты работают в облегченном `Stateless ReAct` цикле. Они не видят основную память агента, его переписки или системный лог.
* **Параллелизм:** Главный агент может запустить сразу нескольких субагентов и спокойно уйти в сон. Субагенты будут работать в фоне (каждый в своей `asyncio` таске) и, когда закончат, пришлют отчеты. Событие `SUBAGENT_TASK_COMPLETED` мгновенно разбудит оркестратора для приемки работы.

### Доступные роли (RBAC)
Каждая роль имеет свой узкий системный промпт и жестко ограниченный доступ к интерфейсам:
* 💻 **CODER**: Доступ к изолированной песочнице (в пределах `sandbox/`), выполнению безопасных скриптов и локальному Git (clone, checkout, commit, push).
* 🕵️ **WEB RESEARCHER**: Доступ к DuckDuckGo, Tavily, Jina, Trafilatura и мощному навыку `DeepResearch` (параллельный поиск и чтение до 20 уникальных веб-страниц за раз).
* 🗄️ **ARCHIVIST**: Доступ к долгосрочной памяти (Vector DB). Отвечает за фоновую ревизию, поиск дубликатов, сжатие фактов и удаление информационного шума для идеальной работы RAG.
* 🛡️ **QA ENGINEER**: Доступ к файловой системе, выполнению кода и сети. Не пишет фичи, а пишет суровые `pytest` тесты, прогоняет их в несколько итераций и ищет краевые случаи.

## ⚙️ Ключевые особенности

* **Тотальный контроль доступа:** Управление полномочиями агента осуществляется в пару кликов через CLI-меню. Вы детально настраиваете, к чему у агента есть доступ: от включения отдельных интерфейсов (Telegram, GitHub) до глубины вмешательства. Права к Host OS варьируются от запертой песочницы (`SANDBOX`) до полного `ROOT`-доступа. А мета-права к конфигурации самой системы - от безопасного минимума (`SAFE`) до прав на самомодификацию (`CREATOR`). Автоматическая защита файлов `.env` встроена на уровне кода.
* **Agentic Introspection (Code Graph):** Агент не слеп. Он может алгоритмически проиндексировать любую кодовую базу (включая собственную архитектуру JAWL), превратив ее в векторно-графовую карту. Навыки семантического поиска и вычисления зависимостей (Trace Dependencies) позволяют ему работать с проектами любого объема, перемещаясь по коду как в IDE и не переполняя контекстное окно.
* **Безопасная самомодификация (Deploy Sessions):** Если агент получает доступ уровня `OPERATOR` (права на изменение исходного кода JAWL), система защищает себя от фатальных поломок. Для изменения архитектуры агент обязан открыть *Deploy Session*. Система автоматически прогоняет тесты и синтаксис-чек; при ошибке происходит автоматический откат (Rollback).
* **Динамический контекст (Custom Dashboards):** Агент может создавать собственные "приборные панели" в системном промпте для своих задач. Фоновые демоны (скрипты в песочнице) могут тихо обновлять эти данные, позволяя агенту видеть актуальные метрики или цены без пробуждения системы и лишних API-запросов.
* **Мощная тестовая база:** Архитектура фреймворка покрыта надежными автотестами (более 450+ тестов). Включает интеграционные тесты баз данных и **End-to-End (E2E) тестирования ReAct-цикла**, гарантирующие хорошую стабильность кода и интерфейсов даже в условиях, когда автономный ИИ-агент самостоятельно переписывает системные файлы в рамках *Deploy Sessions*.
* **Саморасширяемость:** Имея наивысший мета-доступ (`CREATOR`), агент способен сам написать Python-скрипт в песочнице и "на лету" зарегистрировать его функции как свои нативные навыки (`execute_skill`). Ему не нужно ждать обновлений от разработчика - он может сам написать себе коннектор к любой API и тут же начать им пользоваться.
* **Пуленепробиваемый парсинг:** Система адаптирована под открытые и нестабильные LLM. Все вызовы функций проходят через **Pydantic Guard Layer**. Если модель проигнорирует нативный `tool_calls` API и выведет JSON текстом, в Markdown-блоке ` ```json ` или вперемешку с рассуждениями - ядро `ReactLoop` автоматически найдет, извлечет и распарсит полезную нагрузку через трехступенчатый фоллбэк.
* **Smart Bootstrapper:** Забудьте про ручную возню с окружениями. Скрипт `jawl.py` сам создаст `venv`, установит зависимости и запустит CLI-интерфейс. Если вы случайно удалите зависимость, система сама восстановит ее при следующем запуске.
* **Event-Driven Heartbeat:** Агент спит между тиками, но мгновенно просыпается при входящих событиях через `EventBus`. Оставшееся время сна динамически сокращается в зависимости от уровня события (CRITICAL, HIGH, MEDIUM, LOW, BACKGROUND).
* **Drives (Мотиваторы):** Агент обладает математически просчитываемой мотивацией. При отсутствии внешних стимулов, растущий дефицит драйвов (например, "Любопытство") заставит агента проактивно искать работу.
* **API Key Rotator:** Автоматическая ротация пула LLM-ключей. Прозрачно обходит Rate Limits (429) и перманентные баны (401), переключая HTTP-сессии на лету.
* **Interactive CLI & DB Manager:** Встроенный терминальный UI для мониторинга логов, настройки интерфейсов и прямого управления памятью агента (SQL и Vector БД).
* **Отличная документация:** Фреймворк снабжен исчерпывающей документацией (папка `docs/`) и эталонными удобными шаблонами.


## 🎭 Кастомизация личности (Промпт-инженерия)
JAWL позволяет предельно гибко настраивать характер и поведение агента. В директории `src/l3_agent/prompt/personality/` лежат базовые Markdown-файлы с системным промптом (`SOUL.md` и `EXAMPLES_OF_STYLE.md`).

Сборщик промптов (`PromptBuilder`) **автоматически склеивает все `.md` файлы** из этой папки перед отправкой контекста. Это означает, что вы не ограничены текущими файлами. Вы можете создавать сколько угодно собственных документов для описания агента. Например:
* `COMMUNICATION_STYLE.md` - для жесткого контроля тональности и формата ответов.
* `LORE.md` - для загрузки предыстории или правил вашей компании.
* `TABOOS.md` - для описания того, что агенту категорически запрещено говорить.

*При первом запуске система автоматически создаст базовую личность из файлов `.example.md`, если вы не создали их сами.*


## 🚀 Установка и запуск

*Требования: Убедитесь, что у вас установлен Python 3.10+ и он добавлен в PATH.*

1. **Клонирование:**
   ```bash
   git clone https://github.com/th0r3nt/JAWL.git
   cd JAWL
   ```

2. **Конфигурация:**
   * Скопируйте `.env.example` в `.env` и добавьте ваши LLM API ключи (и токены интерфейсов, если планируете их использовать). Впрочем, если вы забудете, CLI-установщик спросит ключ при первом запуске.
   * При необходимости отредактируйте `config/settings.yaml` (настройки модели, БД, лимитов) и `config/interfaces.yaml` (модули и уровни доступа).

3. **Магия одной команды:**
   ```bash
   python jawl.py
   ```
   *Скрипт автоматически развернет виртуальное окружение, скачает библиотеки и откроет интерактивное меню настройки и управления агентом.*


> **🐧 Инструкция для чистого Linux (Ubuntu/Debian VPS):**
> 1. Установите системные зависимости: `sudo apt install -y python3 python3-venv git tmux`
> 2. Обязательно добавьте Swap-файл (минимум 2GB), чтобы система не убила векторную БД при инициализации.
> 3. Запускайте фреймворк внутри `tmux`, чтобы агент продолжил работать после закрытия SSH-сессии: 
>    `tmux new -s jawl` -> `python3 jawl.py`


## 📜 Принципы разработки
* **KISS & DRY:** Код пишется так, чтобы его можно было читать без словаря.
* **No overengineering:** Абстракции вводятся только там, где они решают реальную проблему. Инквизиция ООП нас за это не осудит.
* **SOLID:** Строгая изоляция слоев. Мозг (L3) не знает, как работает база данных (L1), он общается с ней только через четко очерченные интерфейсы.


## 🛡️ Безопасность и Отказ от ответственности
JAWL - это фреймворк, который (при определенных настройках) способен скачивать файлы из сети, писать код и выполнять shell-команды. "С большой силой приходит большая ответственность".

* **Фреймворк поставляется «как есть» (As Is).** Автор не несет ответственности за любые деструктивные действия агента, потерю данных или утечку токенов. Вы запускаете проект на свой страх и риск.
* **Изоляция:** По умолчанию агент работает в режиме `SANDBOX` (доступ только к папке `sandbox/`). **Рекомендовано не включать** уровень доступа `ROOT` (3) на вашей основной рабочей машине. Если вы хотите дать агенту полный доступ к ОС - рекомендуется использовать виртуальную машину (VMBox) или изолированный Linux-контейнер без `sudo`.
* **Ограничения in-process "песочницы":** `sandbox_runner.py` / `rpc_wrapper.py` - это **best-effort in-process barrier**, а не настоящая изоляция. Он способен блокировать тривиальные векторы (`subprocess`, `os.system`, стандартный `open()`, `pathlib`, `os.open`, `_io.FileIO`, `os.fork`/`execv*`/`posix_spawn`, `ctypes.CDLL("libc.*")`, `importlib.reload` защищённых модулей, `os.kill` родительского процесса) и скрабит переменные окружения с секретами. **Он не способен** защитить от мотивированного атакующего: любой Cython/C-extension, нетривиальное использование `ctypes` (загрузка произвольной `.so` мимо `libc`), `mmap`, вызов syscall через inline-assembly в расширениях CPython - обходят in-process guard. Для агента, выполняющего недоверенный Python-код (в т. ч. сгенерированный LLM), используйте внешнюю изоляцию: seccomp / Docker / Podman / отдельный namespace-user / полноценный VM.
* **Секреты и .env:** Никогда не добавляйте в `.env` свои основные, личные или корпоративные токены. Рекомендуется:
  * При включенном GitHub интерфейсе - Отдельный GitHub PAT (Personal Access Token) с минимально необходимыми правами.
  * При включенном Email интерфейсе - специально выделенный почтовый ящик.
  * Ключи LLM с жестко заданными финансовыми лимитами (чтобы агент, случайно уйдя в бесконечный цикл из-за ошибки, не сжег вам кредитную карту).
